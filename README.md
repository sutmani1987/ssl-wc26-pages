# ssl-wc26-pages

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
仓库内容不针对任何特定网站或域名，仅作为一组静态 HTML 文件的集中存放与版本管理。

## 目录说明

```
ssl-wc26-pages/
├── docs/               # 页面源文件存放目录
│   └── *.html          # 独立的 HTML 页面
├── assets/             # 公共资源（CSS、JS、图片等，可选）
├── README.md           # 本文件
└── .gitignore          # Git 忽略规则
```

- `docs/` 目录为主要页面存放位置，每个 HTML 文件均为独立页面。
- `assets/` 目录用于存放页面间共享的静态资源，可根据需要自行组织子目录。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面之间的链接、样式、脚本均应使用相对路径，确保在本地或 GitHub Pages 等静态托管环境中正常工作。
- 仓库不保证页面内容与任何外部资源的兼容性或时效性，仅提供原始文件归档。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/ssl-wc26-pages.git
   ```
2. 直接在浏览器中打开 `docs/` 目录下的 HTML 文件即可预览。
3. 若需部署到 GitHub Pages，可将 `docs/` 设置为发布源。

- 请保持目录结构清晰，避免文件冗余。
